02 · Truth Lab

เห็นกับตา:
E บวก แต่ยังขาดทุนได้

ทดลองเปลี่ยนเส้นทางราคาแล้วเทียบ 5 เรื่องคนละตัว: gross P&L ของ constant-dollar, log baseline, Jensen gap, buy-and-hold และผลหลังต้นทุนจำลอง

THEORETICAL · FULL REBALANCE EVERY STEP

A = C Σrᵢ · R = C ln(Pₙ/P₀)
E = A − R = C Σ[rᵢ − ln(1+rᵢ)] ≥ 0

เมื่อราคาเป็นบวก, C คงที่, rebalance เต็มทุกช่วง, fractional fill ได้ และยังไม่หัก friction สำหรับ return เล็ก E ≈ ½CΣrᵢ²

Net D · เหนือ/ต่ำกว่า B—หักต้นทุน strategy
เงินเติมสูงสุดที่ต้องใช้—จาก cash ledger จำลอง

ตัวอย่างสุดขั้วที่เปิดโปงความเข้าใจผิด: ถ้าราคาจาก 100 เข้าใกล้ศูนย์ A เข้าใกล้ −C แต่ R มุ่งสู่ −∞ จน E มุ่งสู่ +∞ — E ยิ่งใหญ่ในขณะที่ wealth พัง จึงห้ามใช้ E ตัวเดียวตัดสินคุณภาพกลยุทธ์
03 · Published Secrets

“ความลับ” 8 ข้อ
ที่โลกสถาบันเปิดเผยแล้ว

นี่ไม่ใช่สูตร proprietary ของกองทุนใด แต่เป็นหลักร่วมที่เห็นซ้ำในงานของ Bridgewater, AQR, Man Group, งานวิจัย และแนวควบคุมระบบ algorithmic trading

01

Signal ≠ Edge

กฎ BUY/SELL เป็นเพียงเครื่องสร้างการตัดสินใจ Edge ต้องมีเหตุผลเชิงเศรษฐศาสตร์ มีเงื่อนไขล้มเหลว และเหลือบวกหลังต้นทุน

บททดสอบ: เขียน mechanism, benchmark และ falsification rule ก่อนรัน backtest
02

Rebalancing คุม risk ก่อนสร้าง return

AQR สรุปตรง ๆ ว่า rebalancing ไม่ใช่ผู้ชนะที่แน่นอน; ผลต่อ risk คาดการณ์ได้มากกว่าผลต่อ return และความถี่ต้องแลกกับ momentum/cost

บททดสอบ: เทียบ buy-and-hold, schedule เดิม, no-DNA และ net-of-costs AQR ↗
03

Size ด้วย risk ไม่ใช่ดอลลาร์

FIX_C เท่ากันในสินทรัพย์คนละตัวให้ volatility และ tail loss ไม่เท่ากัน กรอบที่ดีกว่าดู volatility, stop distance, liquidity, cash และ portfolio risk รวม

บททดสอบ: position ต้องผ่าน risk cap ทุกตัว ไม่ใช่เลือกสูตรเดียว; Kelly สอนเรื่อง log-growth ไม่ได้ให้สิทธิ์เชื่อ estimate แบบเต็มส่วน Kelly ↗
04

กระจาย “ตัวขับความเสี่ยง”

Bridgewater วางสินทรัพย์ตาม growth/inflation surprises; Man เน้น diversification + risk targeting + risk management การมีหลาย ticker ที่ถูกขับด้วย equity beta เดียวกันยังไม่ใช่การกระจาย

บททดสอบ: ดู marginal risk, correlation ในภาวะเครียด และ regime exposure Bridgewater ↗ · Man ↗
05

Execution เป็นส่วนหนึ่งของกลยุทธ์

Last price ไม่ใช่ executable price, order ID ไม่ใช่ fill และ gross edge ที่เล็กกว่าสเปรด/สลิปเพจไม่มีอยู่จริง DIFF จึงควรตอบ cost-benefit ไม่ใช่เลือกด้วยความรู้สึก

บททดสอบ: เก็บ bid/ask, timestamp, fill, implementation shortfall, reject/partial fill ทุก order
06

Survival มาก่อน optimization

ระบบมืออาชีพบล็อก order ที่ใหญ่/แพง/ซ้ำหรือเกินวงเงินก่อนเข้าตลาด และมีวิธี disable ได้เร็ว ไม่รอให้ signal ยอมรับว่าตัวเองผิด

บททดสอบ: hard limits, pilot size, real-time monitor, kill drill SEC ↗ · FINRA ↗
07

Backtest ที่ดีที่สุดมักเป็นผู้ชนะ noise

ยิ่งลอง DNA/seed/mutation มาก โอกาสเลือกโชคยิ่งสูง ต้องเก็บจำนวน trials, ทำ out-of-sample/walk-forward และปรับ Sharpe สำหรับ selection bias

บททดสอบ: PBO/Deflated Sharpe, untouched test set, realistic costs PBO ↗ · DSR ↗
08

Broker ledger คือความจริง

transaction ใน Firestore ป้องกัน step ซ้ำ แต่ไม่รับรอง exactly-once ระหว่างระบบสองฝั่ง Order intent, client ID, lifecycle และ reconciliation ทำให้รู้ว่าเงินจริงอยู่ตรงไหน

บททดสอบ: ทุก fill ย้อนกลับถึง signal/config/commit ได้ และ unexplained reconciliation break = 0
กองทุนเก่งไม่ได้ชนะเพราะ “รู้อนาคต” อย่างเดียว — แต่มักชนะเพราะรู้ว่า ตัวเองไม่รู้อะไร แล้วจำกัดราคาที่ต้องจ่ายเมื่อผิด

06 · Evidence Ladder

เลื่อนระดับด้วยหลักฐาน
ไม่ใช่ด้วย P&L สวย

ระบบปัจจุบัน deploy UAT ได้ แต่ “ความพร้อมเชิงปฏิบัติการ” กับ “หลักฐานเชิงลงทุน” เป็นคนละแกน ต้องผ่าน exit criteria ทั้งคู่ก่อนขยายเงินจริง

L1ฐานปัจจุบัน

Deployable UAT

deploy, scheduler, state, health, auth และ log พื้นฐาน

  • 100 invocation ไม่มี unhandled error
  • state รอด restart/revision
  • config มี type/unit/range/owner
  • runbook + risk register
L2

Safe Single Strategy

true shadow, hard limits, idempotency และ reconcile

  • shadow 1,000 events, order endpoint = 0
  • duplicate/crash injection ผ่าน
  • UAT reconcile 10 sessions, break = 0
  • kill switch + alerts
L3

Controlled Pilot

วงเงินถูก enforce, approval, traceability และ daily ops

  • risk mandate เป็นตัวเลข
  • rollback / kill drill
  • ทุก fill trace ถึง commit/config
  • 20 sessions ไม่มี limit breach
L4

Portfolio OS

หลาย strategy ใช้ risk, execution และ accounting กลาง

  • NAV/reconcile รวมตรง broker
  • risk contribution + correlation
  • cost/capacity per strategy
  • T+1 attribution
L5

Institutional Grade

independent risk/ops, immutable audit, DR และ governance

  • two-person approval
  • risk หยุดระบบได้เอง
  • independent model review
  • regulatory/legal fit ตามเขตจริง
วัน 1–30 · ทำความจริงให้วัดได้
  • freeze การขยายเงินจริง
  • true shadow mode ที่ไม่มี code path ส่ง order
  • actual-fill ledger + config schema
  • secret manager, CI, heartbeat, runbook
  • นิยาม NAV, external flows และ benchmark ก่อน
วัน 31–60 · ปิดช่อง order และ risk
  • order-intent ledger + client ID
  • ACK/PARTIAL/FILL/REJECT/CANCEL state machine
  • broker reconciliation อัตโนมัติ
  • hard limits + stale/liquidity checks
  • timeout/duplicate/crash/holiday tests
วัน 61–90 · Pilot เฉพาะเมื่อ L2 ผ่าน
  • แยก UAT/Prod ชัดเจน
  • risk mandate และวงเงิน hard-enforced
  • เก็บ spread/slippage/shortfall ทุก order
  • daily reconciliation + operator checklist
  • ห้ามเพิ่มทุน/strategy ระหว่าง pilot
ความจริง: 90 วันอาจพิสูจน์ control และ operation ได้ แต่ไม่พอพิสูจน์ alpha ข้ามหลาย regime — promotion ใช้ exit criteria ไม่ใช่วันในปฏิทินหรือกำไรไม่กี่สัปดาห์
07 · Accounting Truth

Dashboard ที่ผู้จัดการทุน
ควรดูจริง

เปลี่ยนจุดศูนย์กลางจากกราฟ E เพียงเส้นเดียว ไปเป็น accounting, benchmark, risk, execution และ operational truth

Account truth

NAV หลังปรับ external flows

ตอบว่าเงินจริงทั้งบัญชีโตหรือหด โดยรวม cash, position, dividends, financing และทุกต้นทุน

Return = ΔNAV adjusted for deposits/withdrawals
Benchmark truth

Net excess vs predeclared benchmark

เปรียบ buy-and-hold, no-DNA, fixed schedule และ cash ด้วยต้นทุน/ช่วงเวลาเดียวกัน

D_net = strategy net P&L − benchmark net P&L
Risk truth

Drawdown / vol / tail / exposure

กำไรโดยไม่บอกความเสี่ยงไม่มีความหมาย ต้องเห็น max drawdown, concentration, beta/factors และ cash-at-risk

Risk budget ≠ FIX_C
Execution truth

Fill, spread, slippage, shortfall

วัดระยะห่างจาก decision price ถึง fill และต้นทุนต่อหน่วย turnover รวม reject/partial fill

Net edge = gross edge − all-in friction
Research truth

OOS + trial-adjusted evidence

รายงาน train/validation/test แยกชัด, จำนวน DNA ที่ลอง, DSR/PBO, turnover และ capacity

Best backtest ≠ expected live Sharpe

แล้วรู้ว่าต้องทำอะไร

คำตอบไม่ใช่ “เปิดหรือปิด bot” แต่คือรู้ว่าหลักฐานระดับใดอนุญาตให้ทำอะไรได้ โดยไม่ข้ามขั้น

หยุดก่อน — ถ้ายังมีข้อใดข้อหนึ่ง

  • ไม่มี true dry-run ที่รับประกันว่าไม่ place order
  • ไม่มี hard max order/position/loss/drawdown
  • ไม่รู้ order accepted แล้ว fill เท่าไร
  • ไม่มี broker reconciliation
  • รายงาน E เป็นกำไรจริงจาก quote ticks
  • cash ที่ต้องเติมเมื่อราคาลงไม่มีเพดาน

เรียนรู้ต่ออย่างปลอดภัย

  • สร้าง actual-fill accounting และ benchmark
  • บันทึกทุก DNA trial และ untouched OOS
  • จำลอง fee/spread/slippage/latency/rounding
  • stress asset crash, trend, gap, halt, outage
  • ทำ duplicate/timeout/partial-fill chaos tests
  • รัน UAT/shadow จน reconciliation break = 0

ขยายได้เมื่อ “หลักฐาน” อนุญาต

  • net edge รอด benchmark และ costs
  • risk mandate เป็นตัวเลขและ hard-enforced
  • live pilot ผ่านจำนวน session ที่กำหนดไว้ล่วงหน้า
  • drawdown/shortfall/limits อยู่ในกรอบ
  • ทุก fill trace และ reconcile ได้
  • independent review/approval ก่อนเพิ่ม risk
ประโยคเดียวจบ: Alpha หาเงิน · Sizing กำหนดความเร็ว · Risk และ Operations ทำให้ไม่ตายก่อน
