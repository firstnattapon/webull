Learning Guide · Quick Start 101

Rebalancing กำไรมาจากไหน — อ่านสามเส้นให้เป็นใน 10 นาที

คู่มือฉบับย่อของ Shannon Demon DNA Bot: หลักการ rebalance สู่มูลค่าคงที่ เครื่องหมายซื้อ/ขาย และวิธีวัดกำไรจากความผันผวนด้วยเส้น Eₙ — ตรงกับตารางและกราฟบนแดชบอร์ดทุกช่อง

Rₙ = Fix_c·ln(Pₙ/P₀) Aₙ = Fix_c·Σ(Pᵢ/Pᵢ₋₁−1) Eₙ = Aₙ − Rₙ = กำไร

บทที่ 1บอททำอะไร — rebalance สู่มูลค่าคงที่

ตั้งเป้าถือหุ้นมูลค่าคงที่ FIX_C ดอลลาร์ตลอดเวลา (เช่น $1,500) ทุก 5 นาทีบอทเช็คมูลค่าปัจจุบันแล้วดันกลับเข้าเป้า:

value_now = จำนวนหุ้น × ราคาล่าสุด

value_now < FIX_C − DIFF   →  BUY   (ถือน้อยไป ซื้อเพิ่ม)
value_now > FIX_C + DIFF   →  SELL  (ถือเยอะไป ขายออก)
อยู่ในช่วง ±DIFF            →  PASS  (ไม่ทำอะไร)

จำนวนหุ้นที่ส่ง order = |FIX_C − value_now| / ราคาล่าสุด

กฎนี้บังคับพฤติกรรมเดียว: ราคาลงถูกบังคับให้ซื้อถูก ราคาขึ้นถูกบังคับให้ขายแพง — เก็บกำไรจากความผันผวน (volatility harvesting) โดยไม่ต้องทำนายทิศทางตลาดเลย

บทที่ 2สามเส้นที่ต้องรู้ — และเส้นไหนคือ “กำไร”

Rₙ · เส้นอ้างอิง Rₙ = Fix_c × ln(Pₙ / P₀)

กระแสเงินสด baseline ที่ “ระดับราคา” อย่างเดียวอธิบายได้ (rebalance ต่อเนื่องเชิงอุดมคติ) ขึ้นลงตามเทรนด์ — ไม่ใช่ฝีมือ

Aₙ · เงินสดสะสมจริง Aₙ = Fix_c × Σ(Pᵢ/Pᵢ₋₁ − 1)

เงินสดสุทธิจากการ rebalance ทุกสเต็ป (+ รับจากขาย, − จ่ายซื้อ) ยังปนเทรนด์อยู่ จึงใช้วัดกำไรตรง ๆ ไม่ได้

Eₙ · กำไร harvest จริง Eₙ = Aₙ − Rₙ ≥ 0

ทุนส่วนเกินเหนือเส้นอ้างอิง — ตัดเส้น Rₙ (ทุน×เทรนด์) ทิ้ง เหลือกำไรจากความผันผวนล้วน ๆ ที่ไม่มีทิศเทรนด์ปน และไม่ลดลงเลย

หลักจำข้อเดียว: อยากรู้กำไร ให้ดู Eₙ — Aₙ คือกระแสเงินสด Rₙ คือ baseline ของทุน×เทรนด์ที่ต้องหักทิ้ง (คณิตศาสตร์รับประกันว่า Eₙ ≥ 0 เสมอ เพราะ x − ln(1+x) ≥ 0 ทุกค่า x)

เทรนด์ ≠ ทุน — สองอย่างนี้มาพันกันใน Rₙ

คำว่า “แยกทุนออก” มักถูกเข้าใจสลับ — ทุนกับเทรนด์เป็นคนละสิ่ง แต่มาเจอกัน (คูณกัน) ในเส้นอ้างอิง Rₙ:

ทุน · เทรนด์ · Rₙ · Eₙ ต่างกันอย่างไร
สิ่งของนิยามหน่วย
ทุนFix_cUSD (คงที่ — ขนาดเงินที่ลงถือ)
เทรนด์ln(Pₙ / P₀)ตัวเลขล้วน (ทิศ/ขนาดที่ราคาวิ่ง)
Rₙ อ้างอิงFix_c × ln(Pₙ/P₀)USD — ทุน “คูณ” เทรนด์
Eₙ กำไรAₙ − RₙUSD — ตัด Rₙ ทิ้ง ไม่มีทิศเทรนด์ปน

ข้อควรระวัง: Eₙ ที่ตัดเทรนด์ออกแล้ว ยังโตตามทุนอยู่ (Eₙ = Fix_c × Σ(rᵢ − ln(1+rᵢ))) — ลงทุนมากก็ harvest ได้มากเป็นสัดส่วน ดังนั้น “แยกทุนออก” หมายถึง (ก) ไม่เอาเงินต้นมาปนในตัวเลขกำไร (Eₙ เริ่มจาก 0) และ (ข) ตัดทิศเทรนด์ทิ้ง — ไม่ได้แปลว่ากำไรไม่เกี่ยวกับขนาดทุนเลย

ภาพเดียวจบ — เงินสดจริงแตกเป็น “เทรนด์ + กำไร” (Sankey)

อ่านเหมือนท่อน้ำ: ความหนา = จำนวนเงิน · เงินสดจริง Aₙ ไหลเข้าจากซ้าย แล้วแตกเป็นสองท่อ — เทรนด์ Rₙ (ท่ออ้วนที่ต้องตัดทิ้ง) กับ กำไรจริง Eₙ (ท่อบางที่เหลือ)

เห็นชัดในภาพเดียว: เงินสดจริง $150 แทบทั้งก้อนคือเทรนด์ (95%) ที่ต้องตัดทิ้ง — กำไร harvest จริงคือท่อทองบาง ๆ แค่ $7.04 (5%) เท่านั้น นี่คือเหตุผลว่าทำไมห้ามอ่าน Aₙ เป็นกำไร ต้องดูที่ Eₙ
เช็คความเข้าใจ — ยืนยันแล้ว

Rₙ = Fix_c × ln(Pₙ/P₀) คือ เส้นต่อเนื่อง (smooth curve) — ใส่ราคาไหนก็ได้ก็ประเมินค่าได้ทันที ไม่ต้องมีประวัติราคาจริง
Aₙ = Fix_c × Σ(Pᵢ/Pᵢ₋₁ − 1) คือ เส้นขั้นบันได (ไม่ต่อเนื่อง) — มีค่าเฉพาะจุดที่ ราคาปัจจุบัน เกิดขึ้นจริงเท่านั้น

ทั้งสองเส้น “เป้าหมาย” เดียวกันคือราคาปัจจุบัน Pₙ — แต่เพราะเส้นหนึ่งเรียบ (สนแค่ปลายทาง) ส่วนอีกเส้นขั้นบันได (สนทุกก้าวที่เดินจริง) พอราคาย้อนมาที่จุดเดิม ทั้งสองเส้นเลย “ไม่ตรงกัน” — ช่องว่างนั้นแหละคือ Eₙ = กำไร

Rₙ เส้นโค้ง — ประเมินได้ทุกราคา Aₙ เส้นทางจริง — มีค่าเฉพาะจุดที่เดินจริง

บทที่ 3เครื่องหมายบนตาราง — สองมุมที่สวนทางกัน

ตารางแดชบอร์ดมีตัวเลขเงิน 2 ตระกูลที่ใช้เครื่องหมายคนละแบบ อ่านผิดคืองงทันที:

เครื่องหมายเงินบนตาราง trade log
คอลัมน์มุมมองเครื่องหมาย +เครื่องหมาย −
ส่วนต่างเป้าหมาย (USD) ปรับพอร์ตเข้าเป้า ต้องซื้อเข้า (ต่ำกว่าเป้า) ต้องขายออก (เกินเป้า)
ΔAₙ / Aₙ (USD) กระแสเงินสด เงินสดรับ — จากการขาย เงินสดจ่าย — จากการซื้อ

สังเกตว่า “ขาย” เป็นลบในมุมปรับพอร์ต แต่เป็นบวกในมุมเงินสด — ถูกทั้งคู่ เพราะตอบคนละคำถาม: คอลัมน์แรกตอบ “ต้องเติมหรือเอาเงินออกเท่าไหร่ให้พอร์ตกลับเข้าเป้า” ส่วน ΔAₙ ตอบ “เงินสดไหลเข้าหรือออกจากกระเป๋าเท่าไหร่”

บทที่ 4กำไรเกิดเมื่อรอบปิด — ต้องจับคู่ซื้อ-ขาย ขาเดียวไม่นับ

ขายขาเดียวตอนราคาขึ้น ไม่สร้างกำไร — ผลรวมเท่ากับถือเฉย ๆ (buy & hold) เป๊ะ กำไร harvest เกิดเมื่อรอบปิด: ขายแพงแล้วซื้อกลับถูก (หรือกลับกัน) จนจำนวนหุ้นกลับมาเท่าเดิมพร้อมเงินสดส่วนเกิน ลองเทียบสองเหตุการณ์ (Fix_c = 1,500):

Aₙ เงินสดสะสม Rₙ เส้นอ้างอิง Eₙ กำไร harvest

เส้น Aₙ (ประ) ทับ Rₙ สนิทในกราฟขวา — ช่องว่างระหว่างสองเส้นนี้แหละคือกำไร ไม่มีช่องว่าง = ไม่มี harvest

บทที่ 5Anchor P₀ ให้ถูก — Eₙ ต้องเริ่มจาก 0 (แยกทุนออก)

เส้นทั้งสามคำนวณจากหน้าต่าง trade log ที่มองเห็น ดังนั้น P₀ ต้องเป็น ราคาแรกในหน้าต่าง — ถ้าตั้ง P₀ ไกลจากราคาจริง (เช่นค่า default 100 ขณะหุ้นอยู่ที่ 314) ระบบจะมองช่วง 100→314 เป็น “หนึ่งสเต็ป” แล้วฉีดเงินปลอมก้อนยักษ์เข้ามา:

บทที่ 6PASS ทุกแถว = ยังไม่มีเทรดจริง

เส้น Aₙ/Rₙ/Eₙ บนแดชบอร์ดคำนวณจาก เส้นราคา เสมือน rebalance ทุก tick แต่แถวที่สถานะ PASS_THRESHOLD คือบอทไม่ได้ส่ง order (ส่วนต่างยังไม่ทะลุ ±DIFF) ดังนั้น:

  • ถ้าหน้าต่างมีแต่ PASS → ตัวเลขทั้งหมดเป็น what-if เชิงทฤษฎี ไม่ใช่เงินที่ทำได้จริง — แดชบอร์ดจะขึ้นการ์ดเตือนพร้อม metric “เทรดจริง (BUY/SELL) = 0”
  • เงินจริงเกิดเฉพาะแถว BUY/SELL ที่ order ถูก fill — และกำไร realize เมื่อคู่ซื้อ-ขายปิดรอบ (บทที่ 4)
  • DIFF คือตัวคุมจังหวะ: แคบไป = เทรดถี่ ค่าธรรมเนียมกิน · กว้างไป = ปล่อยรอบสวิงหลุดมือ ไม่ได้เก็บ harvest

บทที่ 8Quick Start — ตั้งค่าและอ่านแดชบอร์ดตามลำดับ

ตั้งค่าบอท (env vars หลัก)

FIX_C   = 1500      # มูลค่าเป้าหมายที่ถือคงที่ (USD)
P0      = 314.55    # ราคาอ้างอิงตอนเปิดสถานะ → ใช้คำนวณ baseline_pnl
DIFF    = 60        # ช่องว่าง ±USD ก่อนยอมเทรด (คุมความถี่)
DNA_CODE = bypass:100   # ตารางเวลาเทรด (1=เทรด, 0=ข้าม)

ขั้นตอน deploy เต็ม ๆ อยู่ใน doc/QUICKSTART.md ของ repo บอท — เอกสารนี้โฟกัสที่การอ่านผลให้เป็น

อ่านแดชบอร์ดตามลำดับนี้

  1. ดู Eₙ ก่อน — นี่คือกำไร harvest สะสม (เริ่มจาก 0 เสมอ) ถ้าโตขึ้น = เก็บรอบสวิงได้จริง
  2. เช็ค “เทรดจริง (BUY/SELL)” — ถ้าเป็น 0 ตัวเลขทั้งกระดานคือทฤษฎี ยังไม่มีเงินจริง
  3. Aₙ กับ Rₙ ไว้ดูบริบท — สองเส้นห่างกันแค่ไหน = harvest มากแค่นั้น ทับกันสนิท = ราคาวิ่งทางเดียว
  4. คอลัมน์ส่วนต่างเป้าหมาย — เครื่องหมายบอกทิศ: − ต้องขายออก · + ต้องซื้อเข้า (อย่าสับสนกับเครื่องหมายเงินสดของ ΔAₙ)
  5. เส้น Rₙ เต็มประวัติ ตั้งแต่ P₀ จริงของบอท ดูได้จากคอลัมน์ baseline_pnl ที่บันทึกทุกแถว
สรุปทั้งเล่มในหนึ่งบรรทัด: ซื้อถูก-ขายแพงให้ครบคู่ → ช่องว่างเหนือเส้นอ้างอิงคือกำไร → อ่านที่ Eₙ ซึ่งเริ่มจากศูนย์และไม่มีวันถอยหลัง
