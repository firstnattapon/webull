3 Diagnosis

RC1 — บอทไม่ยืนยันการ fill หลัง place (blind) w = 0.65 ▲

  1. ทำไม log ดูเหมือนขายแต่หุ้นไม่ลด? → บอทถือว่า place สำเร็จ (order_id) = เสร็จ
  2. ทำไมถึงพลาด? → ไม่เคยอ่าน filled_quantity/สถานะจริงกลับมา
  3. Manual Test Lab ต่างยังไง? → มีปุ่ม Order detail / Positions ให้ “ยืนยัน” เสมอ

RC2 — เศษหุ้น/สภาพตลาด (หรือ UAT) ทำให้ “รับแต่ไม่ fill” w = 0.35

  1. หุ้นเต็ม fill ได้ · เศษหุ้นบอทไม่ fill → ถูก drop/expire
  2. หรือ endpoint = UAT sandbox ที่ไม่ fill กับตำแหน่งจริง

RC (เดิม): payload ต่างจาก Manual Test Lab — ตัดทิ้ง

ตรวจแล้ว payload เหมือนกันทุกฟิลด์ (order_v3.place_order, entrust_type=QTY) → การจัด payload ให้ตรงไม่เปลี่ยนอะไร
