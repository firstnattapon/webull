2 Problems

P1 — qty คงที่แม้ SELL ออก 3 ครั้ง severity 5/5

Webull รายงานตำแหน่งเดิมทุก tick → ออร์เดอร์ไม่ fill กับตำแหน่งจริง (พฤติกรรม UAT)

P2 — log ORDER_SUBMITTED ไม่บอกว่าเป็น UAT และไม่ยืนยัน order_id severity 5/5

ทำให้ “ออร์เดอร์ที่รับแต่ไม่ fill (UAT)” ดูเหมือน “ขายจริงแต่หุ้นไม่ลด”

P3 — ไม่รู้ว่า deploy อยู่บน environment ไหน severity 4/5

default WEBULL_ENV=uat และไม่มีคำเตือน/บันทึก endpoint

3 Diagnosis

RC1 — รันบน UAT: รับออร์เดอร์แต่ไม่ fill กับตำแหน่งจริง w = 0.60 ▲

  1. ทำไม qty ไม่ขยับ? → ออร์เดอร์ไม่ถูก fill
  2. ทำไมไม่ fill? → endpoint เป็น UAT sandbox (ยืนยัน: submit เศษหุ้น “accepted” แต่ตำแหน่งไม่เปลี่ยน)
  3. ทำไมเป็น UAT? → config.py default WEBULL_ENV=uat

RC2 — log ไม่ยืนยัน + ไม่ระบุ environment w = 0.40

  1. เขียน ORDER_SUBMITTED ตายตัว ไม่เช็ค order_id/สถานะ
  2. ไม่บอกว่า UAT → อ่าน log ไม่รู้ว่าทำไมหุ้นไม่ลด

RC (เดิม): เศษหุ้นถูกปฏิเสธ — ตัดทิ้ง

พิสูจน์แล้วว่าเศษหุ้น 0.05 ถูกรับ (มี order_id) → ไม่ใช่ต้นเหตุ · โค้ดจึงต้อง “ไม่บล็อกเศษหุ้น”
