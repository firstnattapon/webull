3 Diagnosis — รากของปัญหา ปรับน้ำหนักหลังหลักฐานใหม่

RC1 — ออร์เดอร์ “เศษหุ้น” ถูกปฏิเสธ/ไม่ถูกจอง w = 0.45 ▲

  1. ทำไม qty ไม่ขยับ? → ออร์เดอร์ไม่เข้า order book
  2. ทำไมไม่เข้า? → จำนวนเป็นเศษหุ้น ~0.1 · Manual Test Lab ที่ “ผ่าน” ใช้หุ้นเต็ม (1)
  3. เศษหุ้นอาจต้องใช้ notional/มี min หรือไม่รองรับใน venue/บัญชีนี้
หลักฐาน: preview 1 หุ้นสำเร็จ · บอทส่ง 0.10247/0.10363/0.10777 · ยืนยันเหตุผลจริงได้จาก preview หลังเปิด gate

RC2 — โค้ดตี “สำเร็จ” โดยไม่ยืนยัน + ไม่ preview-gate w = 0.35

  1. place_market_order ไม่เช็ค order_id/สถานะ · main.py เขียน ORDER_SUBMITTED ตายตัว
  2. บอทข้าม preview (default off) และไม่เอาผล preview มากั้นการส่ง
  3. ผล: reject ที่ควรเห็นตั้งแต่ preview กลับกลายเป็น “ส่งแล้ว” ลวงตา

RC3 — มองไม่เห็น environment ที่ยิงเข้า w = 0.20 ▼

  1. default UAT · UAT อาจรับ order แต่ตำแหน่งไม่ขยับ
  2. ลดน้ำหนักลงเพราะพิสูจน์แล้วว่า “กลไกส่ง order ทำงาน” — ประเด็นหลักอยู่ที่เศษหุ้น + การยืนยันผล

สมการวัดผล & ประเมินผล

Goal attainment (อัตรารับจริง)A = ออร์เดอร์ที่ accepted / ออร์เดอร์ที่ตั้งใจส่ง (เป้า A → 1)
ความสอดคล้องของตำแหน่งΔqty_actual = qty(n−1) − qty(n) ควรใกล้ จำนวนสั่ง
ความจริงของ logtruth = #(ORDER_SUBMITTED ที่มี order_id จริง) / #(ORDER_SUBMITTED) (เป้า = 1)

🔁 Control loop — เมื่อยังไม่บรรลุเป้าหมาย

สมการควบคุม (error)e_n = Δqty_expected − Δqty_actual
อัตราการลู่เข้าr_n = |e_n| / |e_(n−1)|
